### The 5 Laws of CSS Skeuomorphism

#### 1. The Global Light Source (The 90° Rule)

Every shadow and highlight must agree on exactly where the sun is (assume top-down, 90° angle).

* **Concave objects (The Track):** The *top inner edge* blocks light (casts an inner shadow). The *bottom inner edge* catches light (casts an inner highlight).
* **Convex objects (The Knob):** The *top outer edge* catches light. The *bottom outer edge* curves into shadow.

#### 2. Material Volume (The Death of Solid Colors)

**Solid colors destroy the 3D illusion.** A flat `backgroundColor` makes an object look like a piece of paper. Even matte plastic bends light.

* The **Track** needs a subtle `linear-gradient` fading from dark (top) to light (bottom) to simulate the sloping floor of a hollow well.
* The **Knob** needs a gradient fading from light (top) to dark (bottom) to make it physically domed (convex).

#### 3. The 3-Tier Cast Shadow (Z-Axis Depth)

A realistic drop shadow isn't one blur. It is three distinct layers stacked together:

* **Ambient Occlusion (Contact Shadow):** A very tight, extremely dark shadow exactly where the knob touches the floor. This grounds the object so it doesn't look like it's floating.
* **Umbra (Directional Shadow):** Gives the object its perceived physical height.
* **Penumbra (Ambient Scatter):** A wide, low-opacity spread that simulates room light scattering in the deep well.

#### 4. The Inverse Cut (Micro-Bevels)

When you CNC-mill a hole into a chassis, the microscopic edge of the cut catches light. You must simulate the outer rim rolling *away* from the light (top outer drop shadow) and the bottom outer rim rolling *toward* the light (bottom outer drop highlight).

#### 5. The "Negative Spread" Trick

When lighting a cylinder/puck, standard CSS drop shadows bleed out the left and right sides, ruining the illusion of a top-down light. By adding a **negative 4th value** (spread radius) to the knob's cast shadows, the shadow tapers off gracefully inward, perfectly mimicking 3D geometry.

### The 4 Pillars of the UI Render Pipeline

#### 1. Fragment Shaders (Materials & Sub-Surface Grain)

CSS `background-color` looks like plastic because CSS gradients are mathematically perfect. Real materials have microscopic chaotic imperfections. We inject an invisible `<svg>` into the DOM using `feTurbulence` to calculate physical grain.

* **Anisotropic Metal:** Stretching the noise X/Y frequency creates the "brushed" directional grooves of machined aluminum.
* **Frosted Glass:** High-frequency noise blended via `screen` creates the micro-roughness of an acrylic lens.

#### 2. Raytraced Lighting Engine (Global Illumination)

Static shadows are dead. Real objects react to their environment. We bind a global `pointermove` listener to calculate a normalized X/Y light vector (`-1.0` to `1.0`) from the user's cursor. #### 3. Real Glass Optics (IOR & Caustics)

* **Caustics:** When light passes through a convex glass sphere, it projects a concentrated "burn" of light on the floor beneath it. We simulate this by rendering a highly saturated radial gradient *under* the glass, moving it mathematically inverse to the light source, and applying `mix-blend-mode: color-dodge`.

#### 4. Hooke's Law Kinematics (Mass & Kinetic Deformation)

CSS `transition: ease` is weightless and fake. Real hardware has **mass ($m$)**, **spring tension ($k$)**, and **friction/damping ($c$)**. # The canon of 2D hyper-real skeuomorphic controls

## 1. One light source only

Pick one dominant key light and never betray it.

For example:

* key light: **top-left**
* cavity darkness: **bottom interior**
* cast shadow: **downward / slightly right**
* rim highlight: **top edge / left edge**
* specular hotspot: **upper-left quadrant of knob**

The instant two layers disagree, the illusion dies and the button becomes glossy mashed potatoes.

---

## 2. Build every control from five physical signals

Every good skeuomorphic button, toggle, slider, or panel is made from these:

1. **Base material**

   * the actual body color and broad surface gradient

2. **Bevel / rim**

   * thin highlight on the lit edge
   * thin dark compression on the opposite edge

3. **Cavity occlusion**

   * the “cut into the panel” darkness inside the track or inset well

4. **Contact shadow**

   * the darkest, smallest shadow directly under the knob where it nearly touches the track

5. **Cast shadow**

   * the softer, larger shadow projecting into space

Most bad skeuomorphism confuses **contact shadow** and **cast shadow**.
They are not cousins. They are different species.

---

## 3. Gradients do the material work; shadows do the spatial work

This is the big one.

* Use **gradients** to describe the *surface*.
* Use **shadows** to describe *depth* and *distance*.

That means:

### Track

Should usually have:

* a subtle **top sheen**
* a broad **top-to-bottom darkening**
* a slightly brighter upper rim
* a deeper lower interior

### Knob

Should usually have:

* a broad vertical gradient
* a soft radial hotspot near the key light
* a darker lower edge
* a very thin rim highlight

If shadows are doing all the work, the component gets muddy.

---

## 4. Dark mode is not “same object, inverted”

For dark skeuomorphism:

* highlights should be **smaller and tighter**
* internal shadows should be **deeper but more local**
* midtone separation matters more than absolute contrast
* speculars should be restrained, or the object looks wet/plastic

For light skeuomorphism:

* you can get away with broader soft occlusion
* highlights can be more diffused
* external cast shadows can be slightly wider without turning into sludge

---

## 5. A toggle is not a button with a sliding circle

A believable toggle is two objects:

* a **machined cavity**
* a **floating actuator**

The cavity should feel carved into the panel.
The knob should feel like a separate piece with mass.

That means the **track** needs inset logic, and the **knob** needs object logic.
Right now your code has some of that, but it needs stronger separation.

---

## 6. Motion must reinforce mass

When toggled:

* animate the knob with `transform: translate3d(...)`
* slightly shorten the cast shadow in the “pressed” or “settled” state
* shift the specular hotspot by a hair
* optionally deepen the contact shadow for 100–150ms during movement

# What “real glass caustics” means on the web

Here is the sober physics goblin answer:

## True caustics

Physically, caustics are light focused by reflection/refraction onto another surface. In full rendering, that wants proper light transport. CSS does not provide programmable fragment-light transport; its primitives are layering, blending, filtering, and backdrop effects. So **physically correct caustics are not a CSS problem**—that is an inference from the available APIs, not a magic prohibition. ([MDN Web Docs][4])

## What you can do instead

For UI, you usually want **plausible caustics**, not path-traced cathedral windows.

Use a layered caustic pass:

1. derive a refractive field from the control’s height/normal
2. warp a light texture / procedural interference map
3. focus energy near lower receiving surfaces
4. modulate by transmission, curvature, and key-light direction
5. temporally animate with tiny phase drift

This produces the *feeling* of light concentration under glass toggles, acrylic knobs, gel pills, or liquid UI capsules.

In CSS backend: fake it with background layers and blend modes.
In WebGPU backend: do it properly as a pass.

# 2. The backend contract

Each backend should receive the **same evaluated node graph** and return a render package.

That keeps your React components sane and prevents backend-specific style spaghetti from breeding in the walls.

# 3. Capability split by backend

## CSS backend

Use this for:

* raised buttons
* inset panels
* toggles
* segmented controls
* acrylic-ish pills
* soft polymer surfaces
* modest glass

It relies on multiple background layers, `background-blend-mode`, `mix-blend-mode`, `filter`, `backdrop-filter`, and stacked shadows. Those are the mature, broad compositing primitives. `backdrop-filter` specifically filters content behind a partially transparent element, which is why it is useful for frosted or transmissive-feeling UI. ([MDN Web Docs][2])

## Paint-worklet / procedural backend

Use this for:

* generated brushed metal
* fine grain
* bevel masks
* per-theme microtexture
* static or semi-static normal-like assets
* authored caustic masks for fallback mode

The CSS Painting API lets you generate CSS image values via `paint()`, but MDN marks `paint()` as limited availability rather than Baseline, so it should be a bonus backend, not your load-bearing spine. ([MDN Web Docs][3])

## WebGPU backend

Use this for:

* per-pixel lighting
* dynamic normals from height
* actual environment-band response
* refraction distortion
* glass edge brightening
* animated caustics approximation
* physics-linked highlight motion
* signed-distance shapes and soft masks
* high-fidelity hero controls

WebGPU is explicitly positioned as the successor to WebGL with better alignment to modern GPUs and support for general-purpose GPU computation, which makes it the correct substrate for a real UI surface engine rather than a CSS ornament pile. ([MDN Web Docs][1])

# 4. Render graph canon

This is the real heart of the engine.

## Pass A — shape solve

Generate the control silhouette as:

* rounded rect
* capsule
* ring
* path mask
* SDF composite

## Pass B — height synthesis

Build a scalar height field from:

* base bulge
* bevel
* inset well
* seam ridge
* press depression
* hover swell
* material micro-noise

## Pass C — normals

Derive normals from height.

## Pass D — occlusion

Compute:

* cavity AO
* edge compression
* contact shadow mask
* underside darkening

## Pass E — direct lighting

Evaluate:

* diffuse
* specular
* rim
* fresnel-ish edge lift
* transmission lift
* environment bands

## Pass F — transmission / refraction

For acrylic or glass:

* background sampling
* local distortion from normals
* edge brightening
* optional spectral split kept extremely subtle

## Pass G — caustic approximation

Project a focused light field onto the receiving plane or lower layers. This is not physically correct caustics in the full rendering sense; it is a plausible UI approximation driven by transmission, curvature, and light direction.

## Pass H — post

* bloom trim
* grain
* anti-banding dither
* color grading
* contrast shaping

That graph applies to WebGPU directly and partially compiles down into CSS approximations.

# 6. WebGPU implementation canon

The WebGPU path should be built as a **2.5D deferred-ish surface renderer**, not a fake 3D engine.

## Recommended buffers

* shape parameters uniform/storage buffer
* material parameters uniform/storage buffer
* light rig uniform buffer
* motion state buffer
* pointer state buffer
* height texture
* normal texture
* AO texture
* transmission distortion texture
* optional caustic accumulation texture

## Recommended pipeline layout

* geometry pass for instanced quads or vector tessellated patches
* height/normal generation pass
* lighting pass
* shadow/composite pass
* post pass

Because WebGPU exposes bind group layouts, devices, command encoders, and supported feature queries, you can build this as a real pipeline with capability detection rather than a prayer circle. Query adapter/device features and limits up front and choose profiles accordingly. ([MDN Web Docs][5])

### Shape rendering strategy

Use one of these:

1. **Instanced quads + analytic SDF in shader**
   Best for pills, circles, rounded rects, rings, and simple icon masks.

2. **Vector tessellation**
   Better for custom paths and ornate UI silhouettes.

3. **Hybrid**
   Most practical: SDF for 90% of controls, tessellation only for exotic shapes.

For your use case, analytic SDF-first is the strongest starting doctrine.

# 7. Input and physics canon

You said real animation and physics. Good. That means controls must react to **measured input state**, not just CSS easing.

Pointer Events unify mouse, touch, and pen input, and `PointerEvent` includes channels like pressure. `getCoalescedEvents()` exposes un-coalesced motion points for smoother precise motion handling, and `pointerrawupdate` gives more immediate pointer-change events. Those are exactly the primitives you want for hotspot chasing, tilt response, drag inertia, and high-fidelity interaction. ([MDN Web Docs][6])

# 8. Real glass doctrine

A lot of “glass UI” is fraudulent jelly. The canon should separate **translucency**, **transmission**, **refraction**, and **caustics**.

## CSS glass

Use:

* partial transparency
* `backdrop-filter`
* bright edge rims
* faint inner glow
* environment bands
* subtle underside shadow

This gives convincing frosted/acrylic behavior for mainstream UI. ([MDN Web Docs][7])

## WebGPU glass

Add:

* distorted background sampling driven by normals
* edge brightening as curvature increases
* local light concentration on receiving surfaces
* animated micro-imperfection
* optional chromatic offset kept tiny

## Caustic approximation canon

For UI, a very good approximation is enough:

* derive curvature + transmission mask
* project light energy onto the lower receiving plane
* modulate by key-light direction
* blur anisotropically
* animate slightly with state and motion

This is not full path tracing. It is a controlled fake, but a scientifically respectable fake.

# 9. Performance canon

Do not run cathedral-mode shading on 400 controls in a settings panel unless your plan is to convert laptops into griddles.

## Tiering

* **Tier 0:** plain CSS tokens
* **Tier 1:** enhanced CSS compositing
* **Tier 2:** procedural textures / worklets / canvas assets
* **Tier 3:** WebGPU hero controls
* **Tier 4:** physics-reactive hero surfaces only when focused or visible

## Key platform levers

`OffscreenCanvas` can run rendering off the main thread and is transferable; `content-visibility` can allow the browser to skip rendering work for offscreen content; and Page Visibility lets you pause expensive animation work when the page is hidden. Those three should be part of the engine’s automatic performance policy. ([MDN Web Docs][8])

## Engine policy

* no hero backend for offscreen controls
* degrade from WebGPU to CSS when static
* pause expensive light transport when hidden
* quantize pointer-reactive updates outside focus targets
* bake microtextures once, reuse everywhere

## 1) `surface-engine-core.ts`

```ts
// surface-engine-core.ts
export type ThemeMode = "light" | "dark";
export type SurfaceBackend = "auto" | "css" | "paint" | "webgpu";

export type SurfacePreset =
  | "polymer.soft"
  | "ceramic.gloss"
  | "metal.anodized"
  | "glass.acrylic.soft";

export interface SurfaceNode {
  id: string;
  preset: SurfacePreset;
  shape: "rounded-rect" | "capsule" | "circle";
  width: number;
  height: number;
  hero?: boolean;
}

let didRegister = false;

export function supportsWebGPU(): boolean {
  return typeof navigator !== "undefined" && "gpu" in navigator;
}

export function supportsPaintWorklet(): boolean {
  return typeof CSS !== "undefined" && "paintWorklet" in CSS;
}

export function resolveSurfaceBackend(
  node: SurfaceNode,
  requested: SurfaceBackend = "auto"
): Exclude<SurfaceBackend, "auto"> {
  if (requested !== "auto") return requested;
  if (node.hero && supportsWebGPU()) return "webgpu";
  return "css";
}

export function clamp01(v: number): number {
  return Math.max(0, Math.min(1, v));
}

export function lerp(a: number, b: number, t: number): number {
  return a + (b - a) * t;
}

export function mapRange(
  value: number,
  inMin: number,
  inMax: number,
  outMin: number,
  outMax: number
): number {
  const t = clamp01((value - inMin) / (inMax - inMin || 1));
  return lerp(outMin, outMax, t);
}

export function registerSurfaceProperties(): void {
  if (didRegister) return;
  if (typeof CSS === "undefined" || typeof CSS.registerProperty !== "function") {
    didRegister = true;
    return;
  }

  const props: Array<{
    name: string;
    syntax: string;
    inherits: boolean;
    initialValue: string;
  }> = [
    { name: "--sx-hover", syntax: "<number>", inherits: false, initialValue: "0" },
    { name: "--sx-press", syntax: "<number>", inherits: false, initialValue: "0" },
    { name: "--sx-hotspot-x", syntax: "<percentage>", inherits: false, initialValue: "50%" },
    { name: "--sx-hotspot-y", syntax: "<percentage>", inherits: false, initialValue: "50%" },
    { name: "--sx-travel", syntax: "<number>", inherits: false, initialValue: "0" },
    { name: "--sx-caustic", syntax: "<number>", inherits: false, initialValue: "0" },
    { name: "--sx-tilt-x", syntax: "<number>", inherits: false, initialValue: "0" },
    { name: "--sx-tilt-y", syntax: "<number>", inherits: false, initialValue: "0" },
  ];

  for (const prop of props) {
    try {
      CSS.registerProperty({
        name: prop.name,
        syntax: prop.syntax,
        inherits: prop.inherits,
        initialValue: prop.initialValue,
      });
    } catch {
      // Ignore duplicate registration / unsupported syntax quirks.
    }
  }

  didRegister = true;
}
```

## 2) `surface-engine-motion.ts`

This is the “alive” layer: hover, press, tilt, hotspot, coalesced pointer samples, and reduced-motion damping.

```ts
// surface-engine-motion.ts
import { useEffect, useMemo, useRef, useState } from "react";
import { clamp01, lerp } from "./surface-engine-core";

export type SurfaceSimState = {
  hoverAmount: number;
  pressAmount: number;
  toggleTravel: number;
  tiltX: number;
  tiltY: number;
  hotspotX: number; // 0..1
  hotspotY: number; // 0..1
  causticEnergy: number;
};

type SpringState = { x: number; v: number };

function stepSpring(
  state: SpringState,
  target: number,
  stiffness: number,
  damping: number,
  dt: number
): SpringState {
  const a = stiffness * (target - state.x) - damping * state.v;
  const v = state.v + a * dt;
  const x = state.x + v * dt;
  return { x, v };
}

export function useSurfaceInteraction(checked: boolean) {
  const [sim, setSim] = useState<SurfaceSimState>({
    hoverAmount: 0,
    pressAmount: 0,
    toggleTravel: checked ? 1 : 0,
    tiltX: 0,
    tiltY: 0,
    hotspotX: checked ? 0.72 : 0.28,
    hotspotY: 0.35,
    causticEnergy: checked ? 0.6 : 0.2,
  });

  const hoverSpring = useRef<SpringState>({ x: 0, v: 0 });
  const pressSpring = useRef<SpringState>({ x: 0, v: 0 });
  const travelSpring = useRef<SpringState>({ x: checked ? 1 : 0, v: 0 });
  const tiltXSpring = useRef<SpringState>({ x: 0, v: 0 });
  const tiltYSpring = useRef<SpringState>({ x: 0, v: 0 });
  const hotspotXSpring = useRef<SpringState>({ x: checked ? 0.72 : 0.28, v: 0 });
  const hotspotYSpring = useRef<SpringState>({ x: 0.35, v: 0 });
  const causticSpring = useRef<SpringState>({ x: checked ? 0.6 : 0.2, v: 0 });

  const live = useRef({
    inside: false,
    down: false,
    pointerX: checked ? 0.72 : 0.28,
    pointerY: 0.35,
    pressure: 0,
    tiltX: 0,
    tiltY: 0,
  });

  const reducedMotionRef = useRef(false);

  useEffect(() => {
    if (typeof window === "undefined" || !window.matchMedia) return;
    const media = window.matchMedia("(prefers-reduced-motion: reduce)");
    const update = () => {
      reducedMotionRef.current = media.matches;
    };
    update();
    media.addEventListener?.("change", update);
    return () => media.removeEventListener?.("change", update);
  }, []);

  useEffect(() => {
    let raf = 0;
    let prev = performance.now();

    const frame = (now: number) => {
      const dt = Math.min(0.032, (now - prev) / 1000);
      prev = now;

      const reduced = reducedMotionRef.current;
      const hoverTarget = live.current.inside ? 1 : 0;
      const pressTarget = live.current.down ? 1 : 0;
      const travelTarget = checked ? 1 : 0;
      const targetHotspotX = lerp(travelTarget ? 0.72 : 0.28, live.current.pointerX, 0.28);
      const targetHotspotY = lerp(0.32, live.current.pointerY, 0.35);
      const targetTiltX = live.current.inside ? (live.current.pointerY - 0.5) * -2 : 0;
      const targetTiltY = live.current.inside ? (live.current.pointerX - 0.5) * 2 : 0;
      const targetCaustic = 0.18 + 0.42 * travelTarget + 0.16 * hoverTarget + 0.12 * pressTarget;

      const stiff = reduced ? 220 : 420;
      const damp = reduced ? 34 : 28;

      hoverSpring.current = stepSpring(hoverSpring.current, hoverTarget, stiff, damp, dt);
      pressSpring.current = stepSpring(pressSpring.current, pressTarget, stiff, damp, dt);
      travelSpring.current = stepSpring(travelSpring.current, travelTarget, 360, 26, dt);
      tiltXSpring.current = stepSpring(tiltXSpring.current, targetTiltX, 250, 24, dt);
      tiltYSpring.current = stepSpring(tiltYSpring.current, targetTiltY, 250, 24, dt);
      hotspotXSpring.current = stepSpring(hotspotXSpring.current, targetHotspotX, 320, 30, dt);
      hotspotYSpring.current = stepSpring(hotspotYSpring.current, targetHotspotY, 320, 30, dt);
      causticSpring.current = stepSpring(causticSpring.current, targetCaustic, 220, 26, dt);

      setSim({
        hoverAmount: clamp01(hoverSpring.current.x),
        pressAmount: clamp01(pressSpring.current.x),
        toggleTravel: clamp01(travelSpring.current.x),
        tiltX: tiltXSpring.current.x,
        tiltY: tiltYSpring.current.x,
        hotspotX: clamp01(hotspotXSpring.current.x),
        hotspotY: clamp01(hotspotYSpring.current.x),
        causticEnergy: clamp01(causticSpring.current.x),
      });

      raf = requestAnimationFrame(frame);
    };

    raf = requestAnimationFrame(frame);
    return () => cancelAnimationFrame(raf);
  }, [checked]);

  const bind = useMemo(
    () => ({
      onPointerEnter: () => {
        live.current.inside = true;
      },

      onPointerLeave: () => {
        live.current.inside = false;
        live.current.down = false;
      },

      onPointerDown: (e: React.PointerEvent<HTMLElement>) => {
        live.current.down = true;
        try {
          e.currentTarget.setPointerCapture?.(e.pointerId);
        } catch {}
      },

      onPointerUp: (e: React.PointerEvent<HTMLElement>) => {
        live.current.down = false;
        try {
          e.currentTarget.releasePointerCapture?.(e.pointerId);
        } catch {}
      },

      onPointerMove: (e: React.PointerEvent<HTMLElement>) => {
        const native = e.nativeEvent;
        const samples =
          typeof native.getCoalescedEvents === "function"
            ? native.getCoalescedEvents()
            : [native];

        const last = samples[samples.length - 1] ?? native;
        const rect = e.currentTarget.getBoundingClientRect();
        const x = clamp01((last.clientX - rect.left) / rect.width);
        const y = clamp01((last.clientY - rect.top) / rect.height);

        live.current.pointerX = x;
        live.current.pointerY = y;
        live.current.pressure = "pressure" in last ? last.pressure || 0 : 0;
        live.current.tiltX = "tiltX" in last ? (last as PointerEvent).tiltX || 0 : 0;
        live.current.tiltY = "tiltY" in last ? (last as PointerEvent).tiltY || 0 : 0;
      },
    }),
    []
  );

  return { sim, bind };
}
```

```ts
  knobInner: React.CSSProperties;
};

export function compileGlassToggleCss(params: {
  theme: ThemeMode;
  checked: boolean;
  sim: SurfaceSimState;
  width?: number;
  height?: number;
}): ToggleCssParts {
  const { theme, sim } = params;
  const width = params.width ?? 256;
  const height = params.height ?? 112;

  const dark = theme === "dark";
  const knobSize = height - 8;
  const travel = width - knobSize - 8;
  const knobX = 4 + travel * sim.toggleTravel;

  const hotspotX = `${(sim.hotspotX * 100).toFixed(2)}%`;
  const hotspotY = `${(sim.hotspotY * 100).toFixed(2)}%`;

  const rootRotationX = (-sim.tiltX * 4).toFixed(2);
  const rootRotationY = (sim.tiltY * 6).toFixed(2);
  const hoverLift = lerp(0, 4, sim.hoverAmount);
  const pressInset = lerp(0, 2.5, sim.pressAmount);

  const bg = dark ? "#12151b" : "#edf2f8";
  const rim = dark ? "rgba(255,255,255,0.10)" : "rgba(255,255,255,0.95)";
  const rimSoft = dark ? "rgba(255,255,255,0.05)" : "rgba(255,255,255,0.42)";
  const deep = dark ? "rgba(0,0,0,0.74)" : "rgba(0,0,0,0.26)";
  const cavity = dark ? "rgba(0,0,0,0.56)" : "rgba(0,0,0,0.16)";
  const cast = dark ? "rgba(0,0,0,0.44)" : "rgba(0,0,0,0.18)";
  const spec = dark ? "rgba(255,255,255,0.18)" : "rgba(255,255,255,0.80)";
  const glassTop = dark ? "#2b3545" : "#ffffff";
  const glassMid = dark ? "#1a2230" : "#dbe8f6";
  const glassLow = dark ? "#121824" : "#c5d7ea";
  const activeGlow = dark ? "rgba(98, 178, 255, 0.18)" : "rgba(110, 188, 255, 0.20)";

  return {

        : `0 ${10 + hoverLift}px ${28 + hoverLift * 2}px -14px rgba(0,0,0,0.20)`,

        radial-gradient(80% 60% at ${hotspotX} ${hotspotY}, ${activeGlow} 0%, transparent 42%),
        linear-gradient(180deg, ${dark ? "#202733" : "#eaf2fb"} 0%, ${dark ? "#0f131a" : "#cbd8e6"} 100%)
```

## 4) `surface-engine-webgpu.ts`

This is the **hero backend skeleton**. It draws a fullscreen quad and shades a toggle analytically in the fragment shader: track, knob, specular, cavity, and a fake caustic bloom.

```ts
// surface-engine-webgpu.ts
import type { ThemeMode } from "./surface-engine-core";
import type { SurfaceSimState } from "./surface-engine-motion";

export type WebGPUToggleParams = {
  width: number;
  height: number;
  checked: boolean;
  theme: ThemeMode;
  time: number;
  sim: SurfaceSimState;
};

export class SurfaceWebGPURenderer {
  private device!: GPUDevice;
  private context!: GPUCanvasContext;
  private format!: GPUTextureFormat;
  private pipeline!: GPURenderPipeline;
  private uniformBuffer!: GPUBuffer;
  private bindGroup!: GPUBindGroup;
  private ready = false;

  async init(canvas: HTMLCanvasElement): Promise<void> {
    if (!("gpu" in navigator)) {
      throw new Error("WebGPU not supported");
    }

    const adapter = await navigator.gpu.requestAdapter();
    if (!adapter) throw new Error("No GPU adapter");

    this.device = await adapter.requestDevice();
    const context = canvas.getContext("webgpu");
    if (!context) throw new Error("webgpu canvas context unavailable");

    this.context = context;
    this.format = navigator.gpu.getPreferredCanvasFormat();

    this.context.configure({
      device: this.device,
      format: this.format,
      alphaMode: "premultiplied",
    });

    this.uniformBuffer = this.device.createBuffer({
      size: 16 * 4, // 4 vec4<f32>
      usage: GPUBufferUsage.UNIFORM | GPUBufferUsage.COPY_DST,
    });

    const shader = this.device.createShaderModule({
      code: /* wgsl */ `
struct Uniforms {
  a: vec4f; // width, height, time, checked
  b: vec4f; // hover, press, themeDark, travel
  c: vec4f; // hotspotX, hotspotY, caustic, dpr
  d: vec4f; // reserved
};

@group(0) @binding(0) var<uniform> u: Uniforms;

struct VSOut {
  @builtin(position) position: vec4f,
  @location(0) uv: vec2f,
};

@vertex
fn vs(@builtin(vertex_index) vertexIndex: u32) -> VSOut {
  var pos = array<vec2f, 6>(
    vec2f(-1.0, -1.0),
    vec2f( 1.0, -1.0),
    vec2f(-1.0,  1.0),
    vec2f(-1.0,  1.0),
    vec2f( 1.0, -1.0),
    vec2f( 1.0,  1.0)
  );

  let p = pos[vertexIndex];
  var out: VSOut;
  out.position = vec4f(p, 0.0, 1.0);
  out.uv = p * 0.5 + 0.5;
  return out;
}

fn sdRoundRect(p: vec2f, b: vec2f, r: f32) -> f32 {
  let q = abs(p) - b + vec2f(r, r);
  return length(max(q, vec2f(0.0, 0.0))) + min(max(q.x, q.y), 0.0) - r;
}

fn saturate(x: f32) -> f32 {
  return clamp(x, 0.0, 1.0);
}

fn smoothBand(x: f32, a: f32, b: f32) -> f32 {
  return smoothstep(a, b, x);
}

@fragment
fn fs(in: VSOut) -> @location(0) vec4f {
  let uv = in.uv;
  let checked = u.a.w;
  let hover = u.b.x;
  let press = u.b.y;
  let themeDark = u.b.z;
  let travel = u.b.w;
  let hotspot = u.c.xy;
  let causticEnergy = u.c.z;
  let time = u.a.z;

  let dark = themeDark > 0.5;

  let trackCenter = vec2f(0.5, 0.5);
  let trackHalf = vec2f(0.48, 0.48);
  let trackR = 0.48;
  let trackP = uv - trackCenter;
  let trackSdf = sdRoundRect(trackP, trackHalf, trackR);

  let knobCx = mix(0.28, 0.72, travel);
  let knobCenter = vec2f(knobCx, 0.5 + press * 0.015);
  let knobRadius = 0.42;
  let knobDist = distance(uv, knobCenter) - knobRadius;

  if (trackSdf > 0.0) {
    return vec4f(0.0, 0.0, 0.0, 0.0);
  }

  let lightDir = normalize(vec3f(-0.55, -0.75, 0.8));
  let local = (uv - knobCenter) / knobRadius;
  let r2 = dot(local, local);
  let sphereZ = sqrt(max(0.0, 1.0 - r2));
  let knobNormal = normalize(vec3f(local.x, local.y, sphereZ));

  let trackNormal = normalize(vec3f(0.0, -0.15, 1.0));

  let trackBase = select(
    mix(vec3f(0.92, 0.95, 0.99), vec3f(0.78, 0.84, 0.90), uv.y),
    mix(vec3f(0.12, 0.15, 0.20), vec3f(0.06, 0.08, 0.11), uv.y),
    dark
  );

  let knobBase = select(
    mix(vec3f(1.0, 1.0, 1.0), vec3f(0.78, 0.86, 0.94), local.y * 0.5 + 0.5),
    mix(vec3f(0.20, 0.24, 0.30), vec3f(0.09, 0.12, 0.18), local.y * 0.5 + 0.5),
    dark
  );

  let trackDiffuse = max(dot(trackNormal, lightDir), 0.0);
  let knobDiffuse = max(dot(knobNormal, lightDir), 0.0);

  let eyeDir = vec3f(0.0, 0.0, 1.0);
  let halfVec = normalize(lightDir + eyeDir);
  let knobSpec = pow(max(dot(knobNormal, halfVec), 0.0), mix(40.0, 85.0, 1.0 - press));
  let specBoost = 0.18 + hover * 0.15;

  let hotspotVec = uv - hotspot;
  let hotspotGlow = exp(-dot(hotspotVec, hotspotVec) * 22.0);

  let cavity = smoothstep(-0.02, -0.18, trackSdf) * (0.35 + 0.20 * uv.y);
  let activeGlow = exp(-pow((uv.x - knobCx) * 4.4, 2.0) - pow((uv.y - 0.5) * 3.0, 2.0)) * (0.08 + 0.14 * checked);

  let causticOffset = sin(time * 1.5 + uv.x * 20.0) * 0.003;
  let causticVec = uv - vec2f(knobCx + causticOffset, 0.70);
  let caustic = exp(-pow(causticVec.x * 8.0, 2.0) - pow(causticVec.y * 16.0, 2.0)) * causticEnergy;

  let trackColor =
    trackBase
    + vec3f(trackDiffuse * 0.08)
    + vec3f(0.16, 0.48, 0.95) * activeGlow
    + vec3f(0.18, 0.72, 1.0) * caustic * 0.18
    - vec3f(cavity * 0.18);

  let glassTint = select(vec3f(0.08, 0.14, 0.22), vec3f(0.12, 0.20, 0.28), dark);
  let knobColor =
    knobBase
    + vec3f(knobDiffuse * 0.14)
    + glassTint * hotspotGlow * 0.16
    + vec3f(knobSpec * specBoost)
    + vec3f(0.12, 0.32, 0.75) * hotspotGlow * 0.10;

  let underKnobShadow = exp(-pow((uv.x - knobCx) * 9.0, 2.0) - pow((uv.y - 0.60) * 18.0, 2.0)) * (0.16 + press * 0.08);

  var color = trackColor - vec3f(underKnobShadow);

  if (knobDist < 0.0) {
    color = knobColor;
  }

  return vec4f(color, 1.0);
}
      `,
    });

    this.pipeline = this.device.createRenderPipeline({
      layout: "auto",
      vertex: {
        module: shader,
        entryPoint: "vs",
      },
      fragment: {
        module: shader,
        entryPoint: "fs",
        targets: [{ format: this.format }],
      },
      primitive: {
        topology: "triangle-list",
      },
    });

    this.bindGroup = this.device.createBindGroup({
      layout: this.pipeline.getBindGroupLayout(0),
      entries: [
        {
          binding: 0,
          resource: {
            buffer: this.uniformBuffer,
          },
        },
      ],
    });

    this.ready = true;
  }

  render(canvas: HTMLCanvasElement, params: WebGPUToggleParams): void {
    if (!this.ready) return;

    const dpr = window.devicePixelRatio || 1;
    const targetW = Math.max(1, Math.round(params.width * dpr));
    const targetH = Math.max(1, Math.round(params.height * dpr));

    if (canvas.width !== targetW || canvas.height !== targetH) {
      canvas.width = targetW;
      canvas.height = targetH;
      this.context.configure({
        device: this.device,
        format: this.format,
        alphaMode: "premultiplied",
      });
    }

    const uniform = new Float32Array(16);
    uniform[0] = targetW;
    uniform[1] = targetH;
    uniform[2] = params.time;
    uniform[3] = params.checked ? 1 : 0;

    uniform[4] = params.sim.hoverAmount;
    uniform[5] = params.sim.pressAmount;
    uniform[6] = params.theme === "dark" ? 1 : 0;
    uniform[7] = params.sim.toggleTravel;

    uniform[8] = params.sim.hotspotX;
    uniform[9] = params.sim.hotspotY;
    uniform[10] = params.sim.causticEnergy;
    uniform[11] = dpr;

    this.device.queue.writeBuffer(this.uniformBuffer, 0, uniform);

    const encoder = this.device.createCommandEncoder();
    const pass = encoder.beginRenderPass({
      colorAttachments: [
        {
          view: this.context.getCurrentTexture().createView(),
          clearValue: { r: 0, g: 0, b: 0, a: 0 },
          loadOp: "clear",
          storeOp: "store",
        },
      ],
    });

    pass.setPipeline(this.pipeline);
    pass.setBindGroup(0, this.bindGroup);
    pass.draw(6, 1, 0, 0);
    pass.end();

    this.device.queue.submit([encoder.finish()]);
  }
}
```

## 5) `SkeuShaderToggle.tsx`

This is the public component. It auto-resolves backend and uses the same interaction model either way.

```tsx
// SkeuShaderToggle.tsx
import React, { useEffect, useMemo, useRef, useState } from "react";
import {
  registerSurfaceProperties,
  resolveSurfaceBackend,
  type SurfaceBackend,
  type SurfaceNode,
  type ThemeMode,
} from "./surface-engine-core";
import { useSurfaceInteraction } from "./surface-engine-motion";
import { compileGlassToggleCss } from "./surface-engine-css";
import { SurfaceWebGPURenderer } from "./surface-engine-webgpu";

type Props = {
  checked: boolean;
  onChange: (value: boolean) => void;
  theme?: ThemeMode;
  backend?: SurfaceBackend;
  width?: number;
  height?: number;
  label?: string;
  hero?: boolean;
};

export function SkeuShaderToggle({
  checked,
  onChange,
  theme = "dark",
  backend = "auto",
  width = 256,
  height = 112,
  label = "SkeuShader Toggle",
  hero = true,
}: Props) {
  const { sim, bind } = useSurfaceInteraction(checked);
  const canvasRef = useRef<HTMLCanvasElement | null>(null);
  const rendererRef = useRef<SurfaceWebGPURenderer | null>(null);
  const [gpuReady, setGpuReady] = useState(false);

  const node: SurfaceNode = useMemo(
    () => ({
      id: "toggle",
      preset: "glass.acrylic.soft",
      shape: "capsule",
      width,
      height,
      hero,
    }),
    [width, height, hero]
  );

  const resolved = resolveSurfaceBackend(node, backend);

  useEffect(() => {
    registerSurfaceProperties();
  }, []);

  useEffect(() => {
    if (resolved !== "webgpu") return;
    const canvas = canvasRef.current;
    if (!canvas) return;

    let cancelled = false;

    (async () => {
      try {
        const renderer = new SurfaceWebGPURenderer();
        await renderer.init(canvas);
        if (cancelled) return;
        rendererRef.current = renderer;
        setGpuReady(true);
      } catch {
        setGpuReady(false);
      }
    })();

    return () => {
      cancelled = true;
    };
  }, [resolved]);

  useEffect(() => {
    if (resolved !== "webgpu" || !gpuReady) return;
    const canvas = canvasRef.current;
    const renderer = rendererRef.current;
    if (!canvas || !renderer) return;

    let raf = 0;
    const start = performance.now();

    const frame = (now: number) => {
      renderer.render(canvas, {
        width,
        height,
        checked,
        theme,
        time: (now - start) / 1000,
        sim,
      });
      raf = requestAnimationFrame(frame);
    };

    raf = requestAnimationFrame(frame);
    return () => cancelAnimationFrame(raf);
  }, [resolved, gpuReady, width, height, checked, theme, sim]);

  const handleClick = () => onChange(!checked);

  if (resolved === "webgpu" && gpuReady) {
    return (
      <button
        type="button"
        aria-pressed={checked}
        aria-label={label}
        onClick={handleClick}
        style={{
          width,
          height,
          padding: 0,
          border: "none",
          background: "transparent",
          cursor: "pointer",
          borderRadius: 999,
          overflow: "hidden",
          display: "inline-block",
          WebkitTapHighlightColor: "transparent",
        }}
        {...bind}
      >
        <canvas
          ref={canvasRef}
          style={{
            width: "100%",
            height: "100%",
            display: "block",
            borderRadius: 999,
          }}
        />
      </button>
    );
  }

  const css = compileGlassToggleCss({
    theme,
    checked,
    sim,
    width,
    height,
  });

  return (
    <button
      type="button"
      aria-pressed={checked}
      aria-label={label}
      onClick={handleClick}
      style={css.root}
      {...bind}
    >
      <span style={css.track} />
      <span style={css.well} />
      <span style={css.caustic} />
      <span style={css.knob}>
        <span style={css.knobInner} />
      </span>
    </button>
  );
}
```

# What this gives you right now

This scaffold already locks the important doctrine:

* **one interaction model**
* **one material vocabulary**
* **CSS first, WebGPU when hero mode is justified**
* **typed custom properties**
* **pointer-driven hotspot and tilt**
* **caustic-style underside light concentration**
* **reduced-motion-aware spring behavior**

Typed custom properties are available through `CSS.registerProperty()` / `@property`, Pointer Events unify mouse/pen/touch and expose pressure plus coalesced samples, and WebGPU’s entry point is `navigator.gpu` with a `GPUCanvasContext` from `getContext("webgpu")`. ([MDN Web Docs][1])

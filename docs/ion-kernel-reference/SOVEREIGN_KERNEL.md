# SOVEREIGN KERNEL

## Status

This is a **provisional doctrine bridge** for the active root's operating physics.
It names the minimum kernel rules the current system is actually using.

## Kernel Rules

### K1 — Bounded Work Unit

A lawful work unit is bounded enough that another agent can see:

- what was attempted
- what changed
- what remains unresolved

### K2 — Load Order

Before meaningful work:

1. read the governing boot or directive
2. read private source continuity
3. read the task or active artifact set
4. read only the additional routed files actually needed

### K3 — Source Update Loop

After a bounded work unit:

1. update private source continuity
2. update public artifacts as needed
3. emit a signal
4. ensure a fresh session could resume

### K4 — Public Bus

The public coordination bus is:

- `ION/05_context/inbox/`
- `ION/05_context/signals/`
- `ION/06_intelligence/`

### K5 — Projection Discipline

Shared projections are for visibility, not source truth.
If source and projection disagree, source governs.

### K6 — Review and Escalation

Architecture, doctrine, release, and contradiction-sensitive work must remain reviewable.
Independent pressure from Daimon, audit, or Sovereign surfaces is a feature, not noise.

### K7 — Low-Burn Runtime

The current kernel should assume low-burn sequential routing by default.
Parallelism is allowed when it is justified, not assumed because it is imaginable.

### K8 — Adapter Separation

Execution substrate and semantic law are different layers.
Cursor/Codex is the current execution adapter.
Future API-native ION should preserve the semantic law while changing the adapter.
